## Auto-Populate Loan Scenarios — No Button, Fully Automatic

Currently the 3 scenarios are generated only when the LO clicks "Auto-Generate from Lead Data". This plan makes generation happen automatically the first time the Loan Scenarios tab loads for a lead, with no click required.

### Trigger rules (when auto-gen fires)

Inside `LoanScenariosTab.tsx`, right after `load()` returns:

1. Auto-generate ONLY if **all** of these are true:
   - `scenarios.length === 0` (no scenarios exist yet for this lead)
   - Required data is present: a `purchase_price` on `mortgage_profiles` OR a `property_value` on the lead
   - The lead is not flagged as auto-skipped (see #4 below)
2. If required data is missing → show a small inline notice in the empty state ("Add purchase price to auto-generate 3 options") instead of generating. No silent failure, no toast spam.
3. After successful auto-gen → show a single toast "3 scenarios auto-generated from lead data" and a per-card caption "Auto-generated · {date} · editable".
4. **One-shot guarantee:** persist a flag `loan_scenarios_autogenerated_at` on `mortgage_profiles.notes` (JSON field already used). If set, never auto-generate again — even if the LO deletes all 3 scenarios. This prevents an accidental delete from re-creating scenarios the LO intentionally removed.

### Re-generation (still manual, unchanged)

The existing "Regenerate from Lead" button stays. It is the only path to overwrite scenarios after the initial auto-gen. Confirm dialog unchanged.

### Reactive re-trigger (optional, off by default)

Not in scope for this pass. If the LO updates purchase price *after* the auto-skip happened (no data → had data later), they use the manual button. Keeps behavior predictable.

### Implementation surface

- **Edit `src/components/crm/tabs/LoanScenariosTab.tsx`** only:
  - Add a `useEffect` that runs after `load()` completes, checks the 4 conditions above, and calls a new `autoGenerateSilently()` helper (same logic as `autoGenerate()` but without the `confirm()` prompt and without the "replace existing" branch).
  - After insert, write `notes.loan_scenarios_autogenerated_at = now()` back to `mortgage_profiles`.
  - In the empty state, replace the "Auto-Generate from Lead Data" button with either:
    - a spinner ("Generating 3 scenarios…") while auto-gen runs, or
    - the missing-data inline notice if price is absent.
  - Keep the manual "Regenerate from Lead" button in the header when scenarios exist.

- **No new files.** `src/lib/crm/autoScenarios.ts` already returns the 3 rows.
- **No DB migration.** The flag lives in the existing `mortgage_profiles.notes` JSON.
- **No edge function.** Runs client-side on tab mount, same as current button.

### Guardrails

- Auto-gen never overwrites existing scenarios.
- Once auto-gen has run once for a lead, it never runs again automatically (flag in `notes`).
- LO retains full control: edit any field, delete any card, regenerate manually.
- If `buildAutoScenarios` returns warnings, they surface as a single info banner above the cards (not toasts).
